# 요청 및 응답

로봇과 상호작용하기 위한 HTTP 기반 인터페이스를 제공합니다.

## 기본 URL

```
https://{robot-host}/api/v1
```

## 요청 형식

모든 요청은 JSON 형식으로 보내야 하며, `Content-Type: application/json` 헤더를 포함해야 합니다.

```shell
curl -X POST https://{robot-host}/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "type": "patrol",
    "destination": "A-101"
  }'
```

## 응답 형식

모든 응답은 JSON 형식으로 반환됩니다.

### 성공 응답

```json
{
  "success": true,
  "data": {
    "id": "task-12345",
    "status": "created"
  }
}
```

### 오류 응답

```json
{
  "success": false,
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "name is required",
    "details": {}
  }
}
```

## HTTP 요청 메서드

| 메서드   | 용도             |
| -------- | ---------------- |
| `GET`    | 리소스 조회      |
| `POST`   | 리소스 생성      |
| `PATCH`  | 리소스 부분 수정 |
| `DELETE` | 리소스 삭제      |

## 상태 코드

| 코드 | 의미                  | 설명                      |
| ---- | --------------------- | ------------------------- |
| 200  | OK                    | 요청 성공                 |
| 201  | Created               | 리소스 생성 성공          |
| 204  | No Content            | 요청 성공, 응답 본문 없음 |
| 400  | Bad Request           | 잘못된 요청               |
| 401  | Unauthorized          | 인증 실패                 |
| 403  | Forbidden             | 권한 없음                 |
| 404  | Not Found             | 리소스를 찾을 수 없음     |
| 429  | Too Many Requests     | 요청 제한 초과            |
| 500  | Internal Server Error | 서버 오류                 |
